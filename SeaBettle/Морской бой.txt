1. Постановка задачи.
Разработать игру "Морской бой".

2. Описание игры
Два поля 10х10, одно своё, второе - соперника.
В своём поле устанавливаются корабли: 4 клетки - 1 шт, 3 клетки - 2 шт, 2 клетки - 3 шт, 1 клетка - 4 шт.
Корабли могут располагаться горизонтально и вертикально. Расстояние между кораблями не менее 1 клетки.
Корабли не должны пересекаться друг с другом. Предусмотреть возможность расстановки кораблей компьютером.
Ходы делаются по очереди. На поле противника указывается точка выстрела.
Варинаты ответа: "Мимо", "Ранил", "Убил".
При удачном выстреле ход не передаётся.
При ответе "Убил" окружаем уничтоженный корабль недоступными для выстрелами клетками.
Игра продолжается до уничтожения всех кораблей одного из играющих.

3. Разработка
3.1. Viewer (оболочка)
На форме 2 поля 10х10 клеток. Ниже полей показаны корабли. Свои корабли перед началом игры выставляем на игровое поле.
Корабли противника подсвечиваются по мере уничтожения.
Клетки поля противника могут быть в 3-ёх состояниях: свободное поле, доступное для выстрела, поле попадения (окружающее уничтоженный корабль).
Над полями надпись о принадлежности хода.
Правее находится 3 кнопки - "Начать игру", "Сдаться", "Автоматическая расстановка кораблей".
3.2. Model (Модель)
Перечисление статуса выстрела - мимо, ранил, убил, конец боя.
Перечисление статуса координат - пусто, корабль, выстрел, попал.
Перечисление типы кораблей - 4х, 3х, 2х, 1х.
Перечисление направлений размещения корабля: горизонтально, вертикально.
Поле, в котором храняться координаты кораблей противника - двумерный массив, элементами которого являются: 0 - пустое поле, 1 - клетка корабля, 2 - выстрел, 3 - попавший выстрел.
Поле, в котором храняться координаты своих кораблей - двумерный массив, элементами которого являются: 0 - пустое поле, 1 - клетка корабля, 2 - выстрел, 3 - попавший выстрел.
Поле количества неубитых клеток кораблей. При инициализации = 20.
Поле статуса последнего выстрела: попал, не попал.
Поле ранения: "ранил" (true), "не ранил" (false).
Поле координат последнего выстрела (строка из 2-ух цифр).
Метод автоматического заполнения координаты корабля.
Метод автоматического заполнения координат кораблей.
Метод заполнения координат кораблей вручную.
Метод очистки координат кораблей.
Метод удаления координат корабля.
Метод выстрела ПК. 
Метод выстрела игрока.
Метод проверки правильности координат корабля (при расстановке).
3.2.1. Метод выстрела игрока.
Входящие параметры - координаты выстрела (x,y) - строка из 2-ух цифр.
Результат - перечисление статуса выстрела.
При пустом поле координат (статус координат "пусто") возвращаем статус выстрела "мимо".
При не пустом поле координат проверяются точки с координатами x+1, x-1, y+1, y-1 для анализа статуса корабля. 
Если в этих координатах есть "корабль", то возвращаем статус выстрела "ранил", иначе "убил".
Меняется статус координаты в поле координат. 
Уменьшаем счётчик неубытых клеток кораблей. Если счётчик стал = 0, то возвращаем статус выстрела "Конец боя".
3.2.2. Метод выстрела ПК (генерация случайных координат).
Входящих параметров нет. Возвращает строку из двух цифр - координат массива кораблей противника. 
Проверяем статус последнего выстрела. Если статус выстрела "убил", поле ранения переключаем на false. 
Если статус "Мимо" или "Убил" и поле ранения false, то генерируем случайное число, поле ранения "false".
Если статус выстрела "ранил", то поле ранения "true", в массиве координат противника проверяем координаты x-1, x+1, y-1, y+1. Если есть выстрелы по этим координатам, то генерируем координаты в противоположную сторону. Если выстрелов не было (первое попадание), то случайным образом выбираем одно из проверяемых значений. 
Если статус "Мимо" и поле ранения true, то проверяем координаты x-2, y-2, x+2, y+2.
При генерации проверяем полученные координаты, если в массиве элемент с этими координатами не пустое поле, то генерируем новые координаты.
3.2.3. Метод заполнения координат кораблей вручную.
Принимает координаты начала корабля, тип корабля, направление размещения. 
Направление размещения по умолчанию вертикально.
Заполняет массив координат кораблей.
3.2.4. Метод удаления координат корабля.
Принимает координаты начала корабля, тип корабля, направление размещения. 
Направление размещения логическое, по умолчанию false - сверху вниз.
Удаляет из массива координат корабль.
3.2.5. Метод очистки координат кораблей.
Входных параметров нет. Очищает массив координат кораблей.
3.2.6 Метод автоматического заполнения координаты корабля.
Входные параметры: тип корабля, направление.
Случайным образом получает координаты начала корабля. Вызывает метод проверки правильности координат корабля, если координаты соответствуют, то помещаем в поле координат своих кораблей.
3.2.7. Метод автоматического заполнения координат кораблей.
Входных параметров нет. Вызывает метод автоматического заполнения координаты корабля для всех кораблей. Параметр "направление" берётся случайным образом.
3.2.8. Метод проверки правильности координат корабля.
Входные параметры: тип корабля, координаты корабля, направление корабля.
Результат логический. Проверяем соответствие каждой клетки корабля правилам, если не соответствует, то возвращаем false.
